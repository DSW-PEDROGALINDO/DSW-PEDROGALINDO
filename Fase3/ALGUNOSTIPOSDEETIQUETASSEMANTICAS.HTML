UNIVERSIDAD NACIONAL ABIERTA Y A DISTANCIA (UNAD)

En HTML5 las etiquetas adoptadas sirven para indicar qué on y cuál es el sinificado de los elementos que contienen dentro.
Se denominan Etiquetas Semánticas, las cuales en el marco del HTML y de la componsición de un sitio web son de suma importancia ya que posibilitan a motores de búsqueda como Google a indexar los contenidos de un sitio. Además, dichas etiquetas también proporcionan maquetar páginas web mediante la definición del esquema principal del documento.
Las etiquetas semánticas indican qué es el contenido que enmarcan y cuál es su relación con el conglomerado de elementos del archivo HTML.

PRINCIPALES ETIQUETAS SEMÁNTICAS Y USOS

header: Para definir la cabecera de la página.
footer: Para definir el pie de página. Normalmente contiene datos sobre quien ha creado la página, datos del copyright, etc.
nav: Permite definir un elemento de navegación de la página web,como el menú principal o menús secundarios.
section: Se utilizan para encerrar el código correspondiente a una sección genérica dentro de un documento.
aside: Sirve para delimitar todo aquel contenido que no es directamente contenido principal de esa página en concreto. Puede usarse por tanto para todos aquellos elementos secundarios como bloques publicitarios, enlaces externos, citas, un calendario, etc.
article: Se utiliza para definir artículos como noticias, entradas de un blog, etc.

ETIQUETAS DE CONTENIDO MULTIMEDIA

Insertar Audio y Vídeo

Video: Sirven para integrar vídeos en las páginas web de la misma forma que se hace con los archivos de imagen utilizando la etiqueta:video
Audio: Se usa para agregar un archivo de audio
Es importante tener en cuenta que los siguientes atributos son útiles:
Controls: Para el caso de las etiquetas audio o video, este atributo activa controles de reproducción integrados: Reproducción, Pausa, Buscar y ajustar volumen.
Height: Configura el alto del reproductor en píxeles.
Width: Configura el ancho del reproductor en píxeles.
Preload: Establece la capacidad de almacenamiento en búfer requerido para la reproducción.
Autoplay: Si el reproductor tiene el almacenamiento requerido en el búfer, este atributo genera la reproducción automática.
Poster: Corresponde a una imagen que se aparecerá cuando el reproductor no está disponible y también cuando está en proceso de cargar el contenido.
loop: Corresponde a un comando que genera la reproducción repetitiva hasta que se presione el botón pausa.
